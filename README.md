# Music Studio Relational Database

## Project Overview
This repository documents the creation of a relational database using PostgreSQL, designed for a hypothetical music school, the `Shade School of Music`. 

Initially, the school relied on an Excel sheet with individual tabs to track student info, parent info, teacher info, billing, and instrument selection, resulting in data duplication. This relational database was engineered to improve data tracking and optimize administrative workflow efficiency.

### [Phase 1: Schema Design, Constraints, and Input](./01-schema-design-constraints-input/)
* **Objective**: Establish the music school's relational database model and input the values for each table. 

### [Phase 2: Administrative Prompts](./02-administrative-prompts/)
* **Objective**: Display the query solutions used to respond to "tickets" submitted by the music school administration.
